# 开发流程规范

## 1. 团队协作原则

- 先接口契约，后前后端并行开发。
- 后台 CRUD 优先使用 JEECG Online 表单/代码生成，复杂逻辑再手工扩展。
- 移动端核心学习链路用代码实现，运营展示区用配置化 schema 渲染。
- 页面配置只影响展示，不越权修改业务数据。
- 每个功能必须有验收标准和测试用例。

## 2. 分支规范

| 分支 | 用途 |
| --- | --- |
| `main` | 生产稳定分支 |
| `develop` | 日常集成分支 |
| `feature/edu-course` | 功能开发分支 |
| `fix/course-playback-auth` | 缺陷修复分支 |
| `release/v1.0.0` | 发布候选分支 |

提交信息建议：

- `feat: add course publish api`
- `fix: prevent unpublished lesson playback`
- `docs: add app page schema contract`
- `chore: update database migration`

## 3. 环境规范

| 环境 | 用途 | 数据 |
| --- | --- | --- |
| 本地 | 开发自测 | 开发者本地数据 |
| dev | 联调 | 可重置测试数据 |
| test | 测试验收 | 接近生产的测试数据 |
| prod | 生产 | 正式数据 |

页面配置发布必须区分 test 和 prod。测试环境发布不应影响生产环境。

## 4. 数据库规范

- 所有业务表使用 `edu_` 前缀。
- 保留 JEECG 标准审计字段：`create_by`、`create_time`、`update_by`、`update_time`、`del_flag`。
- 字典类字段使用明确枚举值，不直接存中文。
- 涉及用户、课程、页面版本的关键表禁止物理删除，使用逻辑删除或状态流转。
- 数据库变更必须有 SQL 迁移脚本，并随代码提交。
- 页面配置 JSON 存储必须保留完整版本，不覆盖历史版本。

## 5. 接口规范

- 移动端 API 不直接暴露后台实体全部字段。
- 列表接口必须分页。
- 课程、课节、信息内容必须过滤状态，只返回可见数据。
- 播放接口必须校验用户是否有观看权限。
- 页面配置接口必须返回版本号，便于 App 缓存和回滚。
- 错误信息面向用户时要可读，日志中保留详细异常。

## 6. 后台页面开发规范

- 常规管理页：优先 JEECG Online 表单/代码生成。
- 业务操作：发布、下线、授权、回滚等必须增加二次确认。
- 关键操作：发布课程、发布页面、回滚页面、发送消息必须写操作日志。
- 富文本输入必须做 XSS 过滤。
- 文件上传必须限制类型和大小。
- 页面编辑器中的组件配置必须做白名单校验。

## 7. 配置化 UI 发布规范

### 7.1 草稿

- 运营人员可以多次保存草稿。
- 草稿不会影响生产 App。
- 草稿必须通过 schema 校验才能保存。

### 7.2 预览

- 发布前必须预览移动端效果。
- 预览必须使用 App 同协议渲染器。
- 预览需要检查图片、跳转、课程状态、信息状态。

### 7.3 发布

- 发布权限独立控制。
- 发布动作必须生成版本记录。
- 发布后刷新 Redis 缓存。
- 发布后保留上一版本可回滚。

### 7.4 回滚

- 回滚本质是发布一个复制自历史版本的新版本。
- 回滚后应记录来源版本。
- 回滚不删除中间版本。

## 8. 测试规范

### 8.1 后端测试

- 课程发布校验。
- 未授权课程播放拦截。
- 学习进度保存。
- 信息上下架过滤。
- 页面 schema 保存、发布、回滚。
- 页面配置未知组件和非法样式校验。

### 8.2 移动端测试

- 首次隐私协议。
- 登录、Token 过期、退出。
- 首页配置拉取失败降级。
- 首页 schema 更新后刷新展示。
- 视频播放、断点续播、进度上报。
- 我的课程列表。
- 消息未读数和已读状态。

### 8.3 兼容性测试

- 低版本 App 拉取高版本 schema。
- App 断网后使用本地缓存。
- 课程下架后首页课程组件展示。
- 页面组件删除后业务数据仍存在。
- 发布空页面或无组件页面时展示默认空态。

## 9. 发布流程

1. 功能分支开发完成并自测。
2. 合并到 `develop` 联调。
3. 测试环境部署。
4. 产品、运营、测试验收。
5. 创建 `release/v1.0.0`。
6. 数据库迁移脚本预演。
7. 生产发布后执行健康检查。
8. 后台发布首页默认配置。
9. App 验证首页、课程播放、消息、我的课程。

## 10. 上线前检查清单

- 隐私协议可查看并记录用户同意。
- App API 全部 HTTPS。
- 后台账号权限已分配。
- 默认首页 schema 已发布。
- 默认课程分类已配置。
- 至少一门课程可正常播放。
- 信息发布和下线正常。
- 页面配置可回滚。
- 数据库备份策略已配置。
- 错误日志可查询。
