# 数据模型与接口定义

## 1. 接口规范

### 1.1 基础约定

- 移动端接口前缀：`/api/app/edu/v1`
- 后台接口前缀：`/api/admin/edu/v1`
- 鉴权方式：`X-Access-Token: <token>`
- 时间格式：`yyyy-MM-dd HH:mm:ss`
- 分页参数：`pageNo`、`pageSize`
- 响应结构沿用 JEECG `Result` 风格。

### 1.2 统一响应

```json
{
  "success": true,
  "code": 200,
  "message": "操作成功",
  "result": {},
  "timestamp": 1770000000000
}
```

### 1.3 分页响应

```json
{
  "records": [],
  "total": 100,
  "size": 10,
  "current": 1,
  "pages": 10
}
```

## 2. 核心数据表

| 字段 | 说明 |

| id | 主键 |

| sort_no | 排序 |

### 2.2 讲师 `edu_teacher`

| --- | --- |
| id | 主键 |

## 3. 移动端接口

### 3.1 登录

#### 发送验证码

`POST /api/app/edu/v1/auth/sms-code`

请求：

```json
{
  "phone": "13200006054"
}
```

响应：

```json
{
  "success": true,
  "code": 200,
  "message": "验证码已发送",
  "result": true
}
```

#### 手机号登录

`POST /api/app/edu/v1/auth/login-by-sms`

请求：

```json
{
  "phone": "13200006054",
  "smsCode": "123456",
  "privacyAgreementVersion": "2026-05-01"
}
```

响应：

```json
{
  "token": "jwt-token",
  "user": {
    "id": "user_001",
    "nickname": "高途用户",
    "avatar": "https://cdn.example.com/avatar.png",
    "phoneMasked": "132****6054"
  }
}
```

## 4. 后台接口

后台 CRUD 可通过 JEECG Online 表单/代码生成生成基础接口，以下只列自定义业务接口。

### 4.1 课程发布

`POST /api/admin/edu/v1/courses/{courseId}/publish`

发布校验：

- 课程标题、封面、分类必填。
- 至少有一个已发布课节。
- 课节绑定的视频必须转码成功或可播放。

### 4.2 课程下线

`POST /api/admin/edu/v1/courses/{courseId}/offline`

### 4.3 授权用户课程

`POST /api/admin/edu/v1/courses/{courseId}/grant-users`

请求：

```json
{
  "userIds": ["user_001", "user_002"]
}
```

### 4.4 页面草稿保存

`POST /api/admin/edu/v1/pages/{pageCode}/drafts`

请求：

```json
{
  "schemaVersion": "1.0.0",
  "minAppVersion": "1.0.0",
  "config": {
    "theme": {},
    "components": []
  }
}
```

### 4.5 页面预览

`GET /api/admin/edu/v1/pages/{pageCode}/versions/{versionNo}/preview`

### 4.6 页面发布

`POST /api/admin/edu/v1/pages/{pageCode}/versions/{versionNo}/publish`

### 4.7 页面回滚

`POST /api/admin/edu/v1/pages/{pageCode}/versions/{versionNo}/rollback`

## 5. 错误码建议

| code | 说明 |
| --- | --- |
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录或 Token 失效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 状态冲突，如课程已下线 |
| 422 | 业务校验失败 |
| 500 | 服务异常 |

业务错误示例：

```json
{
  "success": false,
  "code": 422,
  "message": "课程未授权，无法播放",
  "result": null
}
```
