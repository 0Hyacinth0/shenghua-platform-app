# 教育类移动应用 MVP 方案文档

## 1. 文档目标

本目录用于沉淀教育类移动应用首期建设方案。方案基于参考 App 的核心业务场景做降级设计，优先满足当前立项阶段的三类核心需求：

- 用户可以看视频课程、进入课程学习。
- 运营/教务可以在后台发布课程和信息内容。
- 非技术人员可以在 JEECG 后台通过页面编辑能力调整部分 App UI，并让变更持久化、发布后在生产环境稳定渲染。

本方案面向产品、后端、前端、测试和运维团队，作为后续需求拆分、接口开发、联调验收和上线计划的基础。

## 2. 设计边界

首期不是复刻参考 App。参考 App 包含 AI 学习、题库、商城、社区、复杂推荐、学习计划、客服、物流和会员体系等能力；当前项目只做最小可用闭环：

- 必做：手机号登录、隐私协议、首页内容、课程列表、课程详情、视频播放、我的课程、学习进度、信息发布、系统消息、个人中心基础信息。
- 必做：后台课程管理、视频资源管理、信息发布、用户/报名管理、页面配置发布。
- 暂缓：AI 问答、AI 伴学、题库刷题、商城订单、社区圈子、直播、优惠券、积分、物流、复杂推荐算法。
- 禁止首期扩散：不要把所有 App 页面都做成低代码。低代码 UI 配置只覆盖首页运营区、专题页、活动页、信息聚合页等内容型页面；核心学习链路仍由前端代码保证体验和稳定性。

## 4. 关键假设

- 后端采用 JEECG Boot，并以业务模块方式扩展教育业务。
- 后台管理端沿用 JEECG Vue3 管理端能力，利用 Online 表单/代码生成提升 CRUD 效率。
- 移动端由前端团队自行开发，可以是 UniApp、Flutter、React Native 或原生壳 + H5；本文推荐 UniApp/Vue3 作为首期优先方案，便于与 JEECG 生态配合。
- 视频文件不建议直接由应用服务器承载，推荐使用云点播/对象存储 + CDN，后端只管理视频元数据和播放授权。
- JEECG 的门户设计属于官方高级能力，官方文档标注可设计 PC 和移动端门户、支持组件拖拽和样式设置；如当前授权版本不包含该能力，应在 JEECG 后台内实现同等 schema 编辑器作为兼容替代。

## 5. 官方资料参考

- JEECG 项目介绍：官方说明当前技术栈包含 Vue3.5、Vite6、Spring Boot 3.5、Spring Cloud Alibaba、MyBatis-Plus、Shiro/SAS/Sa-Token，并提供 Online 表单、表单设计、门户设计等能力。
- JEECG 门户设计：官方说明门户支持 PC/移动端个性化样式设计、组件拖拽调整位置、样式和内容配置、移动端独立样式设置。
- JEECG Online 代码生成：官方说明 Online 表单可在线建表并生成前后端 CRUD 代码。
- JEECG APP 代码生成：官方说明 APP 代码生成仅支持列表和表单页面，其他页面需自行开发。
