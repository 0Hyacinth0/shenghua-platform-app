# 阶段性开发计划

## 1. 阶段总览

| 阶段 | 周期 | 目标 | 主要交付 |
| --- | --- | --- | --- |
| M0 立项准备 | 3-5 天 | 明确范围和基础架构 | 需求确认、技术方案、接口草案、原型范围 |
| M1 后端基础与后台 CRUD | 1-2 周 | 建立教育业务基础数据 | 课程分类、讲师、课程、课节、信息管理 |
| M2 App 学习闭环 | 2 周 | 用户能登录、看课、记录进度 | 登录、课程列表、详情、播放、我的课程 |
| M3 信息与消息 | 1 周 | 跑通内容发布和消息提醒 | 信息列表/详情、系统消息、课程通知 |
| M4 JEECG 页面编辑与配置渲染 | 2 周 | 非技术人员可配置首页并发布 | 页面编辑、schema 持久化、App 渲染、回滚 |
| M5 联调验收与试运行 | 1-2 周 | 达到可上线标准 | 测试修复、性能检查、生产部署、运营培训 |

总周期建议 7-10 周。若团队人手少，可先发布 M1-M3 的固定首页版本，再在后续小版本补 M4 页面编辑能力。

## 2. M0 立项准备

### 目标

统一 MVP 范围，确认 JEECG 版本、授权、移动端技术栈和视频服务方案。

### 任务

- 确认 JEECG Boot 版本和部署方式。
- 验证当前 JEECG 是否包含门户设计/页面编辑能力。
- 确定移动端框架。
- 确定视频存储和播放服务：云点播或对象存储。
- 确认短信服务供应商。
- 根据本文档评审功能边界。

### 交付物

- 技术选型确认表。
- 接口定义第一版。
- 数据库表设计第一版。
- 首页和课程页低保真原型。

### 验收标准

- 团队确认首期只做视频课、上课、信息发布和配置化首页。
- 页面编辑器实现路径明确：直接用 JEECG 门户设计或自定义 JEECG 后台编辑器。

## 3. M1 后端基础与后台 CRUD

### 目标

后台可以维护课程基础数据和信息内容。

### 任务

- 新建 `jeecg-module-edu`。
- 创建课程分类、讲师、课程、课节、视频、信息表。
- 使用 JEECG Online 表单/代码生成生成基础 CRUD。
- 补充课程发布/下线业务接口。
- 补充视频资源绑定逻辑。
- 补充信息发布/下线逻辑。
- 配置后台菜单和角色权限。

### 交付物

- 后台课程管理页面。
- 后台课节管理页面。
- 后台信息发布页面。
- 后台视频资源管理页面。
- 后端接口文档。

### 验收标准

- 后台可以创建一门课程并添加课节。
- 后台可以绑定视频资源。
- 后台可以发布和下线课程。
- 后台可以发布和下线信息。

## 4. M2 App 学习闭环

### 目标

学员可以在 App 中完成登录、查看课程、播放视频、继续学习。

### 任务

- 实现隐私协议弹窗。
- 实现手机号验证码登录。
- 实现首页固定版基础展示。
- 实现课程分类和课程列表。
- 实现课程详情和课节目录。
- 实现课节播放页面。
- 实现学习进度上报和断点续播。
- 实现我的课程。

### 交付物

- App 登录流程。
- App 课程学习流程。
- 学习进度 API 联调完成。

### 验收标准

- 新用户可以登录。
- 用户可以进入课程详情。
- 用户可以播放已授权课程视频。
- 用户退出再进入同一课节，可以从上次位置继续。

## 5. M3 信息与消息

### 目标

运营可以发布信息，学员可以在 App 查看通知。

### 任务

- 实现信息列表和详情。
- 实现消息分类概览。
- 实现系统消息列表。
- 实现课程消息列表。
- 实现消息已读。
- 后台支持发送系统消息和课程消息。

### 交付物

- App 信息模块。
- App 消息中心。
- 后台消息管理。

### 验收标准

- 后台发布公告后 App 可见。
- 后台发送消息后 App 未读数更新。
- 用户打开消息后可标记已读。

## 6. M4 JEECG 页面编辑与配置渲染

### 目标

非技术人员可以通过 JEECG 后台页面编辑能力修改首页运营区 UI，并发布到生产 App。

### 任务

- 验证 JEECG 门户设计能力可用性。
- 定义页面 schema JSON 协议。
- 创建 `edu_page`、`edu_page_version`、`edu_page_publish` 表。
- 实现页面草稿保存接口。
- 实现页面预览接口。
- 实现页面发布和回滚接口。
- 在后台编辑器中接入组件白名单。
- 实现 App schema 拉取接口。
- App 实现配置化组件渲染层。
- App 实现本地缓存和失败降级。
- 完成预览、发布、回滚联调。

### 交付物

- JEECG 后台页面编辑入口。
- App 首页配置化渲染。
- 页面版本管理。
- 页面发布和回滚能力。

### 验收标准

- 运营无需改代码即可调整首页组件顺序。
- 运营无需改代码即可修改 Banner、课程推荐、公告等内容。
- 配置保存后可恢复。
- 发布后生产 App 展示对应版本。
- 回滚后 App 展示上一稳定版本。
- App 对未知组件和接口失败有降级处理。

## 7. M5 联调验收与试运行

### 目标

完成上线前质量收敛，进入小范围试运行。

### 任务

- 全链路回归测试。
- 修复高优先级缺陷。
- 压测首页、课程列表、播放信息接口。
- 验证 Redis 缓存刷新。
- 验证数据库备份恢复。
- 配置生产域名、HTTPS、CDN。
- 准备运营培训材料。
- 准备上线回滚预案。

### 交付物

- 测试报告。
- 上线清单。
- 运维部署文档。
- 运营使用手册。

### 验收标准

- P0/P1 缺陷清零。
- 核心接口可用性和响应时间达标。
- 页面配置可发布、可回滚。
- 至少一门课程完整播放验证通过。
- 运营人员可独立完成信息发布和首页配置。

## 8. 二期候选能力

首期稳定后再评估：

- 直播课。
- 题库和练习。
- AI 学习助手。
- 课程评价。
- 收藏和点赞。
- 支付订单。
- 优惠券/会员。
- 社区内容。
- 讲师端。
- 更复杂的推荐算法。
