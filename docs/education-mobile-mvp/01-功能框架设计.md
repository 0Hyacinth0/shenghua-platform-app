# 功能框架设计

## 1. 产品定位

本应用定位为轻量级教育课程 App，服务对象是需要通过移动端观看录播课程、查看课程安排和接收机构信息的学员。首期目标是跑通“内容发布 -> 学员浏览 -> 报名/加入课程 -> 看视频上课 -> 查看通知”的基础闭环。

参考 App 的整体体验可以作为视觉和业务参考，但功能复杂度要明显降低。首期不追求 AI 学习助手、社区、题库和商城，而是把课程内容和信息发布做稳定。

## 2. 用户角色

| 角色 | 说明 | 首期权限 |
| --- | --- | --- |
| 游客 | 未登录用户 | 查看公开首页、公开课程摘要、公开资讯；播放限制由运营配置 |
| 学员 | 已登录用户 | 观看已开放课程、查看我的课程、学习进度、系统消息和课程通知 |
| 运营人员 | 后台内容维护 | 发布信息、配置首页、维护专题、上下架内容 |
| 教务/课程管理员 | 后台课程维护 | 创建课程、课节、视频、讲师、课程分类，管理报名/授权 |
| 系统管理员 | 后台系统维护 | 角色权限、字典、配置、发布审核、数据备份 |

教师端首期不做独立 App。讲师信息先作为课程资料维护；后续如需要教师布置作业、直播或互动答疑，再单独扩展。

## 3. App 端一级导航

推荐首期底部导航保留 4 个入口：

| 导航 | 功能 | 说明 |
| --- | --- | --- |
| 首页 | 课程推荐、公告、资讯、专题入口 | 支持后台低代码配置部分模块 |
| 课程 | 课程分类、课程列表、课程详情、视频学习 | 对应“上课”核心场景 |
| 消息 | 系统通知、课程通知、信息公告 | 首期只做站内消息列表，不做 IM |
| 我的 | 个人信息、我的课程、学习记录、设置 | 简化参考 App 的个人中心 |

不建议首期保留“AI 闪学”独立导航，因为会造成产品预期扩散。若确实需要占位，可作为首页运营位或“学习工具”二期入口。

## 4. App 端功能模块

### 4.1 启动与合规

- 隐私协议弹窗：首次打开展示《用户服务协议》《隐私政策》《儿童隐私保护声明》。
- 登录前限制：未同意隐私协议不得进入 App。
- 个人信息最小化：首期必需个人信息建议只采集手机号、昵称、头像。
- 协议版本记录：用户同意时记录协议版本、时间、设备标识和 IP。

### 4.2 登录与账号

- 手机号验证码登录。
- 退出登录。
- Token 自动续期或过期重新登录。
- 用户资料查看和编辑：头像、昵称、手机号脱敏展示。

首期不做第三方登录、账号密码登录、实名制、会员等级。

### 4.3 首页

首页由固定框架 + 可配置运营区组成：

- 固定框架：顶部搜索入口、课程分类快捷入口、底部导航。
- 可配置运营区：轮播图、公告条、课程卡片组、专题入口、信息流、活动 Banner。
- 数据来源：后台页面编辑器配置布局，组件内部绑定课程、专题或信息列表。
- 发布方式：运营人员在后台编辑草稿，预览通过后发布到 App。

### 4.4 课程学习

- 课程分类：如考研、公务员、职业能力、实用英语等，可由后台字典维护。
- 课程列表：支持分类、关键词、上架状态、免费/报名后可看。
- 课程详情：课程封面、标题、简介、讲师、课节目录、报名/加入课程按钮。
- 课节播放：视频播放器、课节标题、学习进度上报、上一节/下一节。
- 我的课程：展示已报名或已授权课程，标注学习状态和最近学习时间。
- 学习进度：记录每个课节最近播放位置、完成状态、累计学习时长。

首期视频以录播为主，不做直播、弹幕、互动白板、课后作业、考试题库。

### 4.5 信息发布

- 信息分类：公告、活动、学习资讯、政策动态、课程通知。
- 信息列表：支持置顶、推荐、发布时间排序。
- 信息详情：标题、封面、摘要、富文本正文、附件链接。
- 阅读状态：学员已读/未读可选，首期至少支持系统消息已读。

信息发布由后台完成。App 不提供用户发帖或评论，避免变成社区产品。

### 4.6 消息中心

- 系统通知：账号、平台、维护、协议更新。
- 课程通知：课程上新、课节更新、报名成功、开课提醒。
- 活动消息：运营活动、专题推荐。
- 已读处理：列表展示未读数，支持单条已读和全部已读。

首期不做点赞收藏、评论互动、关注提醒、物流消息。

### 4.7 个人中心

- 用户基础信息。
- 我的课程。
- 学习记录。
- 设置：隐私协议、清除缓存、版本号、退出登录。

参考 App 中的订单、购物车、商城、学币、硬件、中奖记录、社区公约等首期不做。

## 5. 后台管理端功能模块

| 模块 | 功能 | 建议实现方式 |
| --- | --- | --- |
| 课程分类管理 | 分类新增、排序、启停 | JEECG Online 表单/代码生成 |
| 讲师管理 | 讲师名称、头像、简介、排序 | JEECG Online 表单/代码生成 |
| 课程管理 | 课程基本信息、上下架、是否公开、封面 | 代码生成 + 手工补充业务校验 |
| 课节管理 | 课节标题、视频、试看、排序、时长 | 代码生成 + 自定义上传/选择视频 |
| 视频资源管理 | 视频文件、播放地址、转码状态、时长 | 自定义业务模块 |
| 报名/授权管理 | 给用户授权课程、取消授权 | 自定义业务模块 |
| 信息发布 | 文章、公告、活动、置顶、发布/下线 | JEECG Online 表单/代码生成 |
| 消息管理 | 系统消息、课程消息、发送对象 | 自定义业务模块 |
| 页面配置 | 首页、专题页、活动页布局配置 | JEECG 页面编辑器/门户设计扩展 |
| 用户管理 | 复用 JEECG 用户与权限体系 | JEECG 内置能力 |

## 6. 核心业务流程

### 6.1 学员看课流程

1. 用户打开 App，同意隐私协议。
2. 用户登录或以游客身份浏览公开内容。
3. 用户进入课程详情。
4. 若课程公开或已授权，用户进入课节播放。
5. App 播放视频，并周期性上报播放进度。
6. 用户回到“我的课程”继续学习。

### 6.2 后台发布课程流程

1. 教务创建课程分类和讲师。
2. 教务创建课程并上传封面。
3. 教务添加课节，绑定视频资源。
4. 教务检查课程目录和播放权限。
5. 课程管理员发布课程。
6. 运营可将课程绑定到首页配置组件。

### 6.3 后台发布信息流程

1. 运营创建信息内容。
2. 选择分类、封面、是否置顶、发布时间。
3. 保存草稿或发布。
4. 发布后 App 信息流和消息中心可展示。

### 6.4 后台配置首页流程

1. 运营进入 JEECG 后台页面编辑器。
2. 选择 App 首页草稿版本。
3. 拖拽组件并调整样式。
4. 为组件绑定课程、信息或专题数据源。
5. 在移动端预览模式检查效果。
6. 提交审核或直接发布。
7. App 获取最新发布版本并渲染。

## 7. 首期验收范围

- 学员能登录并查看首页。
- 学员能打开课程详情并播放视频。
- 学员能在“我的课程”继续学习。
- 后台能创建、编辑、上架、下架课程和课节。
- 后台能发布信息内容，并在 App 展示。
- 后台能配置首页运营区，并在发布后由 App 正确渲染。
- 配置化 UI 变更有版本记录、可回滚、可灰度。
