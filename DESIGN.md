# 教育移动应用 UI 样式设计文档

## 1. 设计目标

本文档只描述移动端 UI 样式与交互规范，依据上传的参考 App 截图提炼视觉语言，并结合本项目首期功能做降级设计。项目排期、功能优先级和实施计划不放在本文档中，统一维护在 `SCHEDULE.md`。

首期 App 的视觉目标：

- 看起来像成熟教育产品，而不是后台系统的移动版。
- 保留参考图中的轻量、亲和、明亮、卡片化、内容流体验。
- 降低功能复杂度，只服务首页、课程学习、消息、我的四个核心 Tab。
- 支持 JEECG 后台页面编辑器配置首页运营区，但 App 端仍保持统一设计规范。

## 2. 参考图风格提炼

### 2.1 首页参考

参考首页呈现以下特征：

- 顶部是“身份选择 + 搜索胶囊 + 辅导入口 + 客服图标”的轻导航结构。
- 分类入口使用 5 列图标宫格，每个图标放在浅色圆角方块中。
- 课程引导卡片采用浅红边框、淡粉背景、红色标签和人物头像。
- 内容区使用大标题、横向卡片流、图文卡片、右侧“换一换/查看更多”操作。
- 底部导航是悬浮胶囊样式，背景半透明，选中项为浅灰圆形底 + 红色图标文字。

### 2.2 我的页参考

参考我的页呈现以下特征：

- 顶部留白较多，头像、昵称和脱敏手机号靠左，签到按钮靠右。
- 快捷入口使用图标 + 文本，图标为彩色 3D/渐变风格。
- 功能分区使用大圆角白色卡片，页面背景为极浅灰。
- 卡片标题加粗，内部功能入口为 4 列网格。
- 底部导航继续使用悬浮胶囊样式，保持全局一致。

### 2.3 本项目取舍

保留：

- 明亮白底、浅灰页面背景。
- 红色作为主品牌色和选中态。
- 大字号标题、圆角卡片、轻阴影。
- 首页内容流和个人中心服务卡片。
- 悬浮胶囊底部导航。

弱化：

- 复杂推荐流、社区圈子、商城、学币、购物车、AI 独立入口。
- 过多彩色运营入口，首期分类控制在 4-8 个。
- 强营销文案和高密度信息。

## 3. 品牌视觉基调

### 3.1 关键词

| 关键词 | 设计含义 |
| --- | --- |
| 清爽 | 大面积留白，模块之间有明确呼吸感 |
| 可信 | 字体厚重、信息层级清楚，按钮状态明确 |
| 亲和 | 使用圆角、浅色图标容器、柔和背景 |
| 聚焦学习 | 首页首屏突出课程、上课、公告，不放商城和复杂社区 |
| 易运营 | 首页运营区组件统一，可由后台配置但不破坏样式 |

### 3.2 视觉方向

整体采用“白底 + 浅灰分区 + 红色强调 + 彩色功能图标”的方向。页面不使用大面积深色背景，不使用复杂渐变作为主视觉，不做营销落地页式大 Hero。

## 5. 字体与字号

移动端默认使用系统字体：

```text
- iOS: PingFang SC / SF Pro
- Android: system-ui / Noto Sans CJK SC
```

字号规范：

| 场景 | 字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| 页面大标题 | 30-34px | 700 | 1.2 |
| 分区标题 | 22-26px | 700 | 1.25 |
| 卡片主标题 | 17-20px | 700 | 1.35 |
| 正文 | 14-16px | 400/500 | 1.5 |
| 辅助信息 | 12-13px | 400 | 1.4 |
| Tab 文案 | 12px | 500/600 | 1.2 |

注意：

- 不使用负字间距。
- 中文标题可以加粗，但不要压得过密。
- 卡片标题最多两行，超出省略。
- 运营卡片上的图片文字由图片承载，页面正文不要叠太多字。

| 首页内容区左右边距 | 16px |
| 卡片内边距 | 14-18px |

### 6.2 圆角

| 组件 | 圆角 |
| --- | --- |
| 搜索框 | 24px |
| 功能图标容器 | 18-22px |

## 7. 图标与插画

### 7.1 图标风格

首页分类和我的页快捷入口建议使用“轻 3D / 渐变图标 + 浅色圆角底”的风格。

要求：

- 同一组图标保持同一视角和体积感。
- 图标主体颜色可以多样，但饱和度不要过高。
- 图标容器尺寸统一。
- 不混用线性图标、实心图标和 3D 图标作为同级入口。

### 7.2 线性图标

底部导航、设置、消息、学习服务内部入口使用线性图标：

- 默认色：`#202331`
- 未选中辅助色：`#5E6370`
- 选中色：`#F0524A`
- 线宽保持 2px 左右。

### 7.3 头像与人物

课程讲师头像、用户头像建议使用圆形或圆角方形。头像不要承担过多营销信息，课程卡片中头像只做增强可信度。

## 8. 全局布局

### 8.1 状态栏和安全区

- 页面顶部预留系统状态栏安全区。
- iOS 底部预留 Home Indicator 安全区。
- 底部悬浮导航不得遮挡核心按钮和列表末尾内容。
- 页面底部内容需要额外 padding，避免被导航盖住。

### 8.2 页面背景

| 页面 | 背景 |
| --- | --- |
| 首页 | `#FFFFFF` 为主，局部分区可用 `#F7F8FB` |
| 课程页 | `#F7F8FB`，课程卡片白底 |
| 消息页 | `#FFFFFF`，列表为主 |
| 我的页 | `#F7F8FB`，卡片白底 |
| 播放页 | 播放区域可深色，其余内容白底 |

## 9. 首页 UI 规范

### 9.1 顶部导航

参考图结构：

```text
[身份/人群下拉] [搜索胶囊 + 吉祥物 + placeholder + 辅导] [客服图标]
```

本项目首期建议：

- 左侧：用户身份或课程分类入口，默认显示“学习”或当前分类。
- 中间：搜索框，文案为“找课程 搜公告”。
- 右侧：消息/客服入口，首期可只保留消息图标。
- 搜索框背景：`#F5F6F8`。
- 搜索框高度：40-44px。
- 搜索框圆角：22px。

### 9.2 分类宫格

首页分类入口可由 JEECG 后台配置，但样式固定：

| 属性 | 规范 |
| --- | --- |
| 列数 | 4 或 5 列；首期推荐 4 列 |
| 图标容器 | 56x56px |
| 容器圆角 | 18px |
| 标题字号 | 14px |
| 行间距 | 18-22px |
| 数量 | 首屏不超过 8 个 |

首期分类建议：

- 免费公开课
- 考研课程
- 职业能力
- 实用英语
- 公告资讯
- 学习资料
- 课程专题
- 为你选课

### 9.3 课程引导卡片

参考图中的浅粉课程卡片可作为首页首屏重点运营位。

规范：

- 背景：`#FFF8F7`
- 描边：`#FFD8D4`
- 圆角：16px
- 内边距：16px
- 左上标签：红色描边小标签
- 标题：18-20px，700
- 副文案：14px，`#9AA0AA`
- 右侧按钮：暖金色或主色，圆角 12-16px

首期用途：

- 推荐一门免费公开课。
- 引导进入课程详情。
- 不显示“添加老师”等强运营文案，改为“立即学习”或“查看课程”。

### 9.4 内容 Tab

参考图有“我的订阅 / 发现 / 圈子”。本项目不做社区圈子，建议改为：

```text
推荐 / 公开课 / 资讯
```

样式：

- Tab 字号：22-26px。
- 激活色：`#202331`。
- 非激活色：`#5E6370`。
- 激活下划线：主色，高 4px，宽 24-32px，圆角 2px。

### 9.5 今日推荐卡片

参考图为三列图文卡片。本项目建议：

- 横向滚动卡片，单卡宽度 30%-36% 屏宽。
- 封面比例：`3:4` 或 `1:1.25`。
- 封面圆角：12px。
- 标题最多两行，字号 16px，字重 600/700。
- 来源/分类文字：12px，灰色。
- 右上操作：“换一换”或“查看更多”，字号 14px，灰色。

### 9.6 免费公开课区块

公开课可以使用横向大卡或双列卡片：

- 标题：24px，700。
- 右侧“查看更多”。
- 卡片封面使用真实课程海报或课程封面。
- 显示报名人数、讲师、课节数时要控制信息密度。

## 11. 底部导航规范

参考图为 5 项底部胶囊导航。本项目首期建议 4 项：

```text
首页 / 课程 / 消息 / 我的
```

如产品坚持保留“AI 闪学”视觉占位，必须标记为二期入口，不进入首期主流程。

样式规范：

| 属性 | 规范 |
| --- | --- |
| 位置 | 底部悬浮，距底部安全区 10-16px |
| 高度 | 64-72px |
| 左右边距 | 16px |
| 背景 | `rgba(255,255,255,0.92)` |
| 圆角 | 32-36px |
| 阴影 | `0 8px 28px rgba(32,35,49,0.10)` |
| 选中底 | `#EEF0F4` 或浅主色 |
| 选中图标/文字 | `#F0524A` |
| 未选中 | `#202331` |

交互：

- 点击 Tab 不做夸张动效。
- 选中态使用圆形或胶囊浅底。
- Tab 文案保持 2-4 个字。

## 12. 卡片规范

### 12.1 普通白卡

- 背景：`#FFFFFF`
- 圆角：16-18px
- 阴影：轻阴影或无阴影
- 描边：可使用 `#EEF0F4`
- 内边距：16px

### 12.2 课程卡

课程卡分为图片卡和横向信息卡。

图片卡：

- 封面比例：3:4。
- 标题两行。
- 底部显示讲师/分类/观看数。

横向卡：

- 左侧封面 112x72px 或 120x80px。
- 右侧标题、讲师、课节数、学习进度。
- 用于课程列表和我的课程。

### 12.3 服务卡

- 标题在卡片左上。
- 内容为 4 列网格。
- 每个入口点击区域至少 56x56px。

## 14. 配置化组件样式约束

JEECG 后台页面编辑器允许运营配置内容，但不能破坏 App 的视觉体系。

允许配置：

- 组件顺序。
- 组件标题。
- 图片、链接、课程/信息数据源。
- 展示数量。
- 上下间距。
- 背景色在白名单内选择。

不允许配置：

- 任意 CSS。
- 任意 JavaScript。
- 超大圆角或超大字体。
- 改变底部导航结构。
- 改变登录、播放器、学习进度等核心业务 UI。

组件默认样式：

| 组件 | 默认表现 |
| --- | --- |
| `banner` | 圆角横幅，左右 16px，支持轮播 |
| `imageNav` | 4 列宫格，浅色图标底 |
| `courseList` | 横向课程卡片流 |
| `infoList` | 资讯列表或三列卡片 |
| `noticeBar` | 浅色公告条 |
| `topicCard` | 大图专题卡 |
| `richText` | 正文内容，不允许脚本 |
| `spacer` | 仅控制垂直间距 |

## 15. 页面空态与异常态

### 15.1 空态

空态使用浅色插画或线性图标，文案简洁：

- 暂无课程
- 暂无消息
- 暂无学习记录
- 页面配置加载失败，已展示默认内容

### 15.2 加载态

- 首页和列表页使用骨架屏。
- 课程播放信息加载使用局部 loading。
- 不使用全屏阻塞 loading，除非是登录或支付类流程。

### 15.3 错误态

- 网络错误：展示重试按钮。
- 页面 schema 错误：展示默认首页。
- 课程不可播放：展示明确原因，如“课程已下线”或“暂无观看权限”。

## 16. 动效原则

- 动效轻，不做游戏化夸张弹跳。
- Tab 切换使用 150-200ms 颜色和背景过渡。
- 卡片点击可有 0.98 缩放反馈。
- 列表刷新使用系统下拉刷新。
- 视频播放页避免复杂动效。

## 17. 适配规范

- 以 375px 宽度为设计基准。
- 使用 rpx 或响应式单位时保持卡片比例稳定。
- iPhone 刘海屏和底部安全区必须适配。
- Android 状态栏高度差异需验证。
- 底部导航浮层不能遮挡最后一行内容。

| 页面 | 样式重点 |

| 消息中心 | 白底列表、分类入口、未读红点 |
| 我的 | 个人头部、快捷入口、学习服务卡、用户服务卡、专题 Banner |

## 19. 设计验收标准

- 首页首屏能明确看到搜索、分类、推荐课程和底部导航。
- 我的页能明确看到用户信息、学习服务和用户服务。
- 页面色彩以白色、浅灰和主红为主，不杂乱。
- 卡片圆角、间距、字号在各页面保持一致。
- 配置化组件发布后仍符合本文档样式边界。
- 底部导航在所有主 Tab 中位置和样式一致。
- 旧版本 App 或 schema 加载失败时不出现白屏。
