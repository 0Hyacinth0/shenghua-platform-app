# 项目功能优先级与排期文档

## 1. 文档目标

本文档单独维护项目功能优先级、阶段排期、里程碑和验收标准。UI 样式设计见 `DESIGN.md`，接口和数据模型见 `docs/education-mobile-mvp/04-数据模型与接口定义.md`。

## 2. 已确认选型

| 层级 | 选型 |
| --- | --- |
| 后端 | 与既有方案一致：JEECG Boot / Spring Boot 3 分支、MyBatis-Plus、MySQL 8、Redis、对象存储或云点播 |
| 后台管理端 | JEECG Vue3 管理端，优先使用 Online 表单、代码生成、权限体系、页面编辑/门户设计能力 |
| 移动端 | UniApp + Vue3 |
| 配置化 UI | JEECG 后台页面编辑器生成 JSON schema，UniApp 端白名单组件渲染 |

## 3. 功能优先级

### 3.1 P0 必须完成

P0 是上线最小闭环，没有这些能力不进入试运行。

| 模块 | 功能 | 说明 |
| --- | --- | --- |
| 合规启动 | 隐私协议弹窗、协议版本记录 | 首次启动必须确认 |
| 账号登录 | 手机号验证码登录、退出登录、Token 过期处理 | 首期只做手机号 |
| 首页基础 | 固定首页框架、Banner、课程推荐、公告入口 | 先支持默认配置 |
| 课程管理后台 | 分类、讲师、课程、课节、视频资源 CRUD | JEECG Online/代码生成优先 |
| 课程发布 | 课程上架、下架、课节排序、视频绑定 | 发布前做完整性校验 |
| App 课程 | 分类列表、课程列表、课程详情、课节目录 | 支撑用户找课 |
| 视频播放 | 获取播放地址、播放页、断点续播、进度上报 | 支撑“能上课” |
| 我的课程 | 已报名/已授权课程、最近学习记录 | 支撑继续学习 |
| 信息发布 | 后台发布公告/资讯/活动，App 列表和详情展示 | 支撑“能发布信息” |
| 消息基础 | 系统消息、课程通知、未读/已读 | 不做 IM 聊天 |
| 页面配置发布 | 首页运营区 schema 拉取、缓存、降级 | 支撑后台可配置 UI |

### 3.2 P1 首期增强

P1 可以随首期一起做，也可以跟随第一个小版本上线。

| 模块 | 功能 | 说明 |
| --- | --- | --- |
| 页面编辑器 | JEECG 页面编辑/门户设计接入、移动端预览 | 非技术人员拖拽调整 |
| 页面版本 | 草稿、发布、历史版本、回滚 | 保证生产可控 |
| 专题页 | 活动页、信息聚合页、课程专题 | 复用 schema 渲染 |
| 搜索 | 课程搜索、信息搜索 | 简单关键词搜索即可 |
| 课程授权 | 后台给用户授权课程、取消授权 | 适合内部运营 |
| 消息发送 | 后台按用户或课程发送消息 | 可先不接推送 |
| 用户标签 | 学段、目标、兴趣标签 | 只做标签，不做复杂算法 |

### 3.3 P2 二期候选

P2 暂不进入首期开发计划。

| 模块 | 功能 |
| --- | --- |
| AI 学习 | AI 问答、AI 伴学、智能推荐 |
| 题库练习 | 刷题、错题本、测评报告 |
| 直播课 | 直播预约、直播间、回放 |
| 商城订单 | 支付、订单、优惠券、会员、积分 |
| 社区互动 | 发帖、评论、点赞、关注 |
| 讲师端 | 老师发布资料、答疑、学员管理 |

## 4. 总体排期

以 2 名后端、1 名前端、1 名测试的最低配置估算，总周期 8 周左右。若 JEECG 门户设计不可用，需要自研轻量页面编辑器，整体增加 1-2 周。

| 周期 | 阶段 | 后端任务 | 前端任务 | 测试/产品任务 | 交付 |
| --- | --- | --- | --- | --- | --- |
| 第 1 周 | M0 立项与基础搭建 | 确认 JEECG 版本、建库、建模块、接口框架 | UniApp 项目初始化、路由、请求封装、Tab 框架 | 评审需求和原型 | 可联调基础框架 |
| 第 2 周 | M1 课程后台 | 分类、讲师、课程、课节、视频 CRUD | 登录、隐私协议、首页静态框架 | 后台用例编写 | 后台能维护课程 |
| 第 3 周 | M2 课程 App | 课程列表、详情、播放授权接口 | 课程列表、详情、播放器、进度上报 | 课程链路测试 | 用户能看视频 |
| 第 4 周 | M3 学习闭环 | 我的课程、学习进度、报名/授权 | 我的课程、断点续播、学习记录 | 学习链路回归 | 学习闭环完成 |
| 第 5 周 | M4 信息消息 | 信息发布、消息、未读/已读 | 信息列表/详情、消息中心 | 信息消息测试 | 内容发布闭环 |
| 第 6 周 | M5 页面配置一期 | 页面 schema 表、发布接口、缓存 | schema 拉取、本地缓存、白名单渲染 | 配置化页面用例 | 首页可配置 |
| 第 7 周 | M6 页面编辑和回滚 | JEECG 页面编辑器接入、预览、回滚 | 预览适配、专题页渲染 | 生产/测试隔离测试 | 非技术人员可配置 UI |
| 第 8 周 | M7 联调验收 | 缺陷修复、日志、性能优化 | 兼容修复、体验打磨 | 全链路验收、上线清单 | 试运行版本 |

## 5. 阶段任务

### 5.1 第 1 周：立项与基础搭建

目标：

- 确认技术版本、仓库结构、基础环境、接口规范。

后端：

- 确认 JEECG Boot 版本和部署方式。
- 新建教育业务模块规划。
- 准备 MySQL、Redis、文件存储配置。
- 确认短信和视频服务供应商。

前端：

- 初始化 UniApp + Vue3 项目。
- 配置 `pages.json`、底部 Tab、全局样式。
- 封装 request、Token、本地缓存工具。
- 搭建首页、课程、消息、我的四个空页面。

验收：

- 前后端项目能启动。
- App 可打开四个 Tab。
- 后端接口文档路径确定。

### 5.2 第 2 周：课程后台与基础 App

目标：

- 后台能维护课程基础数据，App 能完成登录和基础首页。

后端：

- 建立课程分类、讲师、课程、课节、视频资源表。
- 使用 JEECG Online/代码生成生成基础 CRUD。
- 实现课程发布/下线校验。

前端：

- 实现隐私协议弹窗。
- 实现手机号验证码登录。
- 实现首页静态版：顶部搜索、分类入口、推荐课程卡片。

验收：

- 后台可以创建一门课程并添加课节。
- App 可以登录并进入首页。

### 5.3 第 3 周：课程学习链路

目标：

- 用户能从课程列表进入详情并播放视频。

后端：

- 课程分类列表接口。
- 课程列表接口。
- 课程详情接口。
- 课节播放信息接口。
- 播放权限校验。

前端：

- 课程列表页。
- 课程详情页。
- 课节目录。
- 视频播放页。
- 播放器基础控制。

验收：

- 至少一门测试课程可完整播放。
- 未授权课程不能播放。

### 5.4 第 4 周：学习闭环

目标：

- 用户能继续学习，学习进度可恢复。

后端：

- 报名/授权接口。
- 学习进度保存接口。
- 我的课程接口。
- 最近学习时间和完成状态计算。

前端：

- 加入课程。
- 播放进度周期上报。
- 断点续播。
- 我的课程页。
- 学习记录入口。

验收：

- 用户退出再进入同一课节，可从上次位置继续。
- 我的课程能展示学习进度。

### 5.5 第 5 周：信息与消息

目标：

- 运营能发布信息，用户能查看通知。

后端：

- 信息发布、下线、列表、详情。
- 系统消息、课程消息、活动消息。
- 用户消息已读/未读。

前端：

- 信息列表和详情。
- 消息中心。
- 消息分类列表。
- 已读操作。

验收：

- 后台发布公告后 App 可见。
- 消息未读数正确变化。

### 5.6 第 6 周：配置化页面一期

目标：

- App 能拉取页面 schema 并用白名单组件渲染首页运营区。

后端：

- 创建 `edu_page`、`edu_page_version`、`edu_page_publish`。
- 实现已发布页面配置接口。
- 实现缓存刷新。
- 实现 schema 校验。

前端：

- 实现 schema 拉取。
- 实现本地缓存。
- 实现默认首页降级。
- 实现 `banner`、`imageNav`、`courseList`、`infoList`、`noticeBar` 组件。

验收：

- 首页运营区可以由 schema 驱动。
- schema 接口失败时展示默认首页，不白屏。

### 5.7 第 7 周：JEECG 页面编辑器

目标：

- 非技术人员能在后台拖拽配置首页，并发布到 App。

后端/后台：

- 验证并接入 JEECG 门户设计或自定义轻量编辑器。
- 实现草稿保存。
- 实现移动端预览。
- 实现发布和回滚。
- 实现发布权限和操作日志。

前端：

- 适配页面预览渲染。
- 实现专题页 schema 渲染。
- 处理未知组件跳过逻辑。

验收：

- 运营能调整首页组件顺序和内容。
- 发布后 App 正确展示。
- 回滚后 App 展示上一稳定版本。

### 5.8 第 8 周：联调验收与试运行

目标：

- 完成上线前质量收敛。

后端：

- 修复缺陷。
- 增加关键日志。
- 性能优化。
- 数据库备份和生产配置检查。

前端：

- 兼容性修复。
- 真机验证。
- 页面细节打磨。
- 弱网和异常态处理。

测试/产品：

- 全链路回归。
- 输出测试报告。
- 准备上线清单。
- 准备运营培训材料。

验收：

- P0 缺陷清零。
- 课程播放、信息发布、页面配置三条主链路通过验收。
- 可进入小范围试运行。

## 6. 里程碑

| 时间点 | 里程碑 | 验收标准 |
| --- | --- | --- |
| 第 3 周末 | 看课链路可用 | App 能登录、看课程列表、进详情、播放至少一门课程 |
| 第 5 周末 | 内容发布闭环 | 后台能发布信息，App 能展示，消息未读可用 |
| 第 7 周末 | 配置化 UI 可用 | 页面可保存、预览、发布、回滚，App 渲染 schema |
| 第 8 周末 | 试运行版本 | P0 完成，P1 高风险项关闭或延期，核心链路验收通过 |

## 7. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| JEECG 当前版本不包含门户设计能力 | 页面编辑器延期 | 用 JEECG 后台自研轻量 schema 编辑器兜底 |
| 视频服务选型太晚 | 播放链路卡住 | 第 1 周必须确定对象存储或云点播 |
| 页面配置范围过大 | 前端渲染器复杂度失控 | 只开放白名单组件和内容型页面 |
| 参考 App 功能诱导范围扩散 | 排期失控 | P2 功能全部不进入首期 |
| UniApp 视频叠层差异 | 播放页 UI 异常 | 播放器页面减少复杂覆盖层，按平台验证 |
| 后台发布误操作 | 生产 UI 异常 | 草稿、预览、发布权限、回滚必须实现 |
